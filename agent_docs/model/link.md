# Link
> Source: `sefaria/model/link.py`

## Purpose
Defines the `Link` and `LinkSet` models representing bidirectional connections between two textual references (Refs) in the Sefaria corpus. Links are stored in the `links` MongoDB collection and power the connections panel, link explorer, and commentary association features.

## Key Classes

### Link
- **Inherits**: `AbstractMongoRecord`
- **Collection**: `links`
- **Role**: Represents a single bidirectional connection between exactly two Refs. Each link has a type (e.g., "commentary", "quotation", "essay") and may be auto-generated or manually created.
- **Key fields**:
  - `refs` — list of exactly 2 Ref strings. **Always stored in sorted order** for deduplication.
  - `type` — lowercase string describing the connection type; `"essay"` type has special validation requiring `versions` and `displayedText`.
  - `auto` — bool; whether generated by an automated process.
  - `generated_by` — string origin: `"add_commentary_links"`, `"add_links_from_text"`, or `"mishnah_map"`.
  - `expandedRefs0` / `expandedRefs1` — precomputed segment-level expansions of each ref (used for efficient querying).
  - `availableLangs` — list of two lists showing which languages (`"he"`, `"en"`) have content for each ref.
  - `charLevelData` — character/word-level alignment data (used by quotation finder).
  - `score` — quality/accuracy score (used by quotation finder).
  - `inline_reference` — dict for matching inline references (itags).
  - `inline_citation` — bool flag for wrapped-refs logic.
  - `versions` / `displayedText` — required only when `type == "essay"`.
- **Key methods**:
  - `_normalize()` — lowercases `type`, normalizes both refs via `Ref().normal()`, coerces `_id` to `ObjectId`.
  - `_pre_save()` — **sorts `refs` alphabetically** for deterministic storage; performs duplicate detection including section-level vs ranged-ref equivalence; if a duplicate exists, updates it (with score/charLevelData, type, or auto metadata) and raises `DuplicateRecordError` rather than creating a second record; also checks for "more precise" existing links; calls `_set_available_langs()` and `_set_expanded_refs()` unless skipped via private flags.
  - `ref_opposite(from_ref, as_tuple=False)` — given a Ref, returns the other Ref in the link (the one on the opposite side). Uses `from_ref.regex()` for matching, so `from_ref` can be less specific than the stored ref.

### LinkSet
- **Inherits**: `AbstractMongoSet`
- **Collection**: `links`
- **Role**: Queryable set of Link records. Can be initialized with either a query dict or a `Ref` object (uses `expandedRefs0`/`expandedRefs1` regex matching).
- **Key methods**:
  - `__init__(query_or_ref)` — if given a `Ref`, builds an `$or` query across `expandedRefs0` and `expandedRefs1` using `Ref.regex(as_list=True)`.
  - `filter(sources)` — filters the set by text title or category name; expands category names into constituent book titles. Returns a **list** (not a LinkSet).
  - `refs_from(from_ref, as_tuple=False, as_link=False)` — returns the Refs on the opposite side of each link relative to `from_ref`. Supports returning as plain Refs, `(from_ref, opposite_ref)` tuples, or `(Link, opposite_ref)` tuples.
  - `get_first_ref_in_linkset(base_text, dependant_text)` — class method; finds the first (by order_id) link between two texts, excluding `add_links_from_text` generated links.
  - `summary(relative_ref)` — returns counts of linked texts grouped by primary category, relative to a given Ref.

## Non-Obvious Patterns
- **Refs are always sorted alphabetically** before storage. This is the deduplication mechanism: two links with the same refs in different order are considered duplicates. When refs are reversed during sort, `charLevelData`, `versions`, and `displayedText` are also reversed to maintain correspondence.
- **Duplicate saves raise `DuplicateRecordError`** (not silently ignored). The error message describes what was updated. Callers should catch this if re-saving is expected.
- **Section-level / ranged-ref equivalence**: `_pre_save()` checks whether a ranged ref that covers an entire section is equivalent to the section ref itself, and treats them as duplicates.
- **`_override_preciselink` flag**: When set on a Link instance before save, allows overriding a more precise existing link (deletes the old one and saves the new one). Without this flag, saving a less-precise link when a more-precise one exists raises `DuplicateRecordError`.
- **`_skip_lang_check` and `_skip_expanded_refs_set` flags**: Private instance flags that skip the (expensive) language availability lookup or expanded refs computation during save. Used by the title-change cascade to avoid re-checking content that hasn't changed.
- **`expandedRefs0`/`expandedRefs1`** are denormalized fields stored for query performance. `LinkSet.__init__` queries these fields (not `refs`) when initialized with a Ref.

## Relationships
- **Depends on**: `abstract` (base classes), `text` (Ref, Index, VersionSet, library), `sefaria.system.exceptions`, `sefaria.system.database`
- **Depended on by**: `sefaria/helper/text.py`, `sefaria/views.py`, `sefaria/tracker.py`, `sefaria/client/wrapper.py`, `sefaria/search.py`, `sefaria/sheets.py`, `sefaria/helper/topic.py`, `sefaria/helper/linker/`, `sefaria/model/schema.py`, `sefaria/model/linker/`
- **Dependency subscriptions** (from `dependencies.py`):
  - `Index` title change (`attributeChange`, `"title"`) -> `process_index_title_change_in_links()` — rewrites all matching refs, expandedRefs0, and expandedRefs1 in-place (skips lang check and expanded refs recomputation).
  - `Index` delete -> `process_index_delete_in_links()` — deletes all links referencing the deleted index.

## Free Functions

- `process_index_title_change_in_links(indx, **kwargs)` — cascade handler; rewrites old title to new title in `refs`, `expandedRefs0`, `expandedRefs1` across all affected links.
- `process_index_delete_in_links(indx, **kwargs)` — cascade handler; deletes all links whose refs match the deleted index.
- `update_link_language_availabiliy(oref, lang=None, available=None)` — updates `availableLangs` on all links connected to a Ref. If `lang`/`available` provided, sets directly; otherwise re-saves to trigger fresh lookup.
- `get_link_counts(cat1, cat2)` — returns book-to-book link counts between two categories (or individual books). Used by Link Explorer.
- `get_book_link_collection(book, cat)` — returns link data formatted for the Link Explorer frontend (title + location pairs).
- `get_book_category_linkset(book, cat)` — returns a LinkSet of links between a book and a category (or two books).
- `get_category_category_linkset(cat1, cat2)` — returns a LinkSet of links between two categories.
- `get_category_commentator_linkset(cat, commentator)` — returns links between a category's books and a commentator on those books.

## Common Tasks

- **Find all links for a Ref**: `LinkSet(some_ref)` — uses expanded refs for matching.
- **Get the "other side" of links**: `LinkSet(ref).refs_from(ref)` — returns list of opposite Refs.
- **Check link counts between two books/categories**: `get_link_counts("Tanakh", "Bavli")`.
- **Create a new link**: `Link({"refs": ["Genesis 1:1", "Rashi on Genesis 1:1"], "type": "commentary"}).save()` — refs will be sorted and normalized automatically; raises `DuplicateRecordError` if equivalent link exists.
- **Update language availability after text change**: `update_link_language_availabiliy(oref, lang="en", available=True)`.
